1 - Conceitos básicos
A linguagem de programação é uma forma de interligar o usuário com o computador à fim de fazê-los se comunicarem, e permitindo o usuário de usar o computador ao seu favor para realizar tarefas, etc.

As etapas para o desenvolvimento de um programa são:
- Análise: É estudado o enunciado do problema, à fim de definir os dados de entrada, o processamento, e os dados de saída.
- Algoritmo: Ferramentas descritivas, como um fluxograma, são utilizadas para descrever o problema, com suas soluções.
- Codificação: O algoritmo é transformado em códigos de linguagens de programação.

Como construir um algoritmo?
- Compreenda completamente o problema a ser resolvido, destacando os pontos importantes e objetos que o compõe.
- Definir dados de entrada: dados que precisam ser fornecidos e quais objetos fazem parte desse problema.
- Definir o processamento: definir o que deve ser feito à fim de resolver o problema, utilizando os dados de entrada (ou não).
- Definir dados de saída: dados que sairão como resultado de seu processamento e dos dados de entrada nele inseridos.
- Construir um algoritmo com as informações que você obteve: os dados de entrada, o processamento e os dados de saída, através de um dos tipos de algoritmo (que serão citados à frente).

Tipos de algoritmo:
- Descrição narrativa
Escrever o algoritmo utilizando de uma linguagem natural. (Como a língua portuguesa)
Vantagens: Familiaridade com a linguagem, tornando fácil a criação do algoritmo.
Desvantagens: A interpretação pode ser diferente de pessoa para pessoa, ao trocar de linguagem para outra, por exemplo.
- Fluxograma
Escrever o algoritmo por meio de formas e símbolos gráficos pré-definidos.
Vantagens: Fluidez maior na compreensão devido ao uso de imagens.
Desvantagens: Necessidade de aprender a simbologia dos fluxogramas, e o algoritmo resultante acaba sendo difícil de transcrever para programas devido à sua simplicidade.
- Pseudocódigo ou portugol
Escrever, por meio de regras prédefinidas, os passos a serem seguidos para a resolução.
Vantagens: Ao conhecer as palavras reservadas de outra linguagem, a transferência do portugol à outra linguagem de programação é quase imediata.
Desvantagens: Há a necessidade de aprender as regras do pseudocódigo.

Para armazenar informação pré-definida pelo programador, ou então, definida pelo usuário, são utilizadas as variáveis, onde seria a representação de uma informação por meio de um nome dado á ela.
Ex: (Na linguagem de programação PHP) $x = 0;
Nesse caso, a sigla $ é utilizada para mostrar que será introduzida uma variável, o "x" seria o nome dessa variável, e o símbolo de igual representa a conexão entre o nome da variável, e o que ela vai representar, o que neste caso seria o número zero, e então depois, há a sintaxe da linguagem, ";", utilizada para demonstrar o fim da linha de código.

Tipos de dados:
Os tipos de dados mais utilizados são: numéricos, lógicos e literais (ou caracteres):
- Numéricos:
Se dividem em dois grupos, inteiros e reais.
Números inteiros podem ser positivos ou negativos, não tendo parte fracionária. Ex: 200, -50.
Números reais podem ser positivos ou negativos, e possuem parte fracionária. Ex: 200.12, 2.54. (É visto que o dado é armazenado utilizando da norma inglesa, ao invés de serem usadas vírgulas, são usadas pontos)
- Lógicos:
Dados booleanos (oriundos da álgebra de Boole) que assumem valores de verdadeiro ou falso.
- Literais (ou caracteres):
Dados formados por um único caractere, ou uma cadeia de caracteres, que incluem letras maiúsculas, minúsculas, números (que geralmente não são usáveis para cálculos), e caracteres especiais. Ex: "João foi à feira e comprou 2 pães!"

Formação de identificadores:
Identificadores são os nomes das variáveis, constantes, rotinas, unidades, etc.
As regras básicas para formar um identificador são:
- Os caracteres permitidos são letras maiúsculas, minúsculas, e o caractere sublinhado (_).
- O primeiro caractere deve ser uma letra ou o caractere sublinhado.
- Não são permitidos espaços em branco e caracteres especiais.
- Não é permitido usar as palavras reservadas nos identificadores, ou seja, que fazem parte da linguagem de programação.

Linguagem PASCAL:
Foi desenvolvida em 1968 por Niklaus Wirth, feito para a programação científica, e utilizada para estudar a lógica de programação.
Ela possui um ambiente integrado de desenvolvimento chamado Turbo Pascal que:
- Apresenta um editor que permite digitar, salvar e modificar códigos.
- Possui um compilador que converte os códigos dos programas em instruções de máquina, e verifica a existência de erros de sintaxe.
- Possui um depurador que permite a inspeção de um programa durante sua execução.
- Conta com um sistema de ajuda nativo.
- Possui o ambiente de execução propriamente dito, que permite a execução dos programas sem sair do Turbo Pascal, assim como permite a criação de arquivos que rodam fora do ambiente do Turbo Pascal (.exe).

Linguagem C/C++:
Dennis Ritchie inventou a linguagem de programação C e foi o primeiro a implementá-la com um computador que utilizava o sistema operacional Unix.
Foi originada da evolução de linguagens, que começou com a linguagem BCPL, desenvolvida por Martin Richards, que influenciou uma outra linguagem ironicamente denominada B, feita por Ken Thompson.
Na década de 1970, a linguagem de programação B levou ao desenvolvimento da linguagem C.
O padrão de C era o fornecido com a versão 5 no sistema operacional Unix. Porém, devido à popularização de microcomputadores, surgiram muitas variantes da linguagem.
Para resolver a situação, a American National Standards Institute (Ansi) estabeleceu em 1983 um padrão para guiar todas as variantes da linguagem C.
A linguagem C++ não é nada mais do que uma extensão de C, mas com suporte para programação orientada a objetos, com uma sintaxe basicamente idêntica à da C.

Linguagem JAVA:
Essa linguagem de programação tem como foco simplicidade, orientação a objetos, portabilidade, alta performance e segurança. Os programas são escritos em arquivos texto com extensão .java, e ao serem compilados pelo javac, são gerados arquivos .class (constituído por bytecodes, código interpretado pela Máquina Virtual Java).
A plataforma Java é composta pela sua máquina virtual, e o Application Programming Interface (API), composta por amplo conjunto de classes já implementadas e testadas.

2 - Paradigma de programação
Um paradigma de programação é o jeito que o programador fez de resolver um problema, com suas regras, filosofias e metodologias, definindo como o código é estruturado
São existentes vários paradigmas: estruturado, orientado a objetos, lógico, funcional, etc.

- Paradigma estruturado (ou imperativo ou estrutural):
Qualquer problema pode ser quebrado em problemas menores, com soluções mais fáceis, conhecidas como sub-rotinas ou funções.
Sub-rotinas podem receber valores de entrada, executam um processo e retornam valores de saída.
O paradigma estruturado diz que todo processamento pode ser realizado pelo uso de três estruturas: sequencial, condicional e iterativa (de repetição).

- Paradigma orientado à objetos:
O problema é considerado como uma coleção de objetos interagindo por meio de mensagens. Os objetos são estruturas de dados contendo dados e lógica.
Assim, objetos com os mesmos dados e lógica dão origem à uma classe.
O programador, ao usar esse paradigma, analisa o problema tentando identificar objetos que fazem parte dessa realidade, e como eles interagem entre si.
