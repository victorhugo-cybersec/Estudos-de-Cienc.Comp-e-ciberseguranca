---------------- Capítulo 1

--------GCC = Significa GNU Compiler Collection, é um conjunto de compiladores de open-source que foi criado nos anos 80, sendo o compilador padrão de praticamente todos os distros do kernel Linux. No Linux, é utilizado para converter código em C ou C++ em um executável (.exe/.elf).

--------AT&T (GCC AT&T) = Nos anos 70, os laboratórios da empresa de telefonia AT&T (Bell Labs) criaram o sistema operacional Unix e a linguagem C. Lá, criaram um estilo próprio de escrever código Assembly, que funcionava para os processadores Intel, e outras marcas de chips da época. Nessa sintáxe do GCC, o MOV usa a ordem MOV ORIGEM, DESTINO, registradores usam % no início, por exemplo: %eax, números hexadecimais usam $0x no início, ao acessar memória, é utilizado dessa maneira:

--Exemplo: [esp + 4] ficaria 4(%esp)

--------Ao criar uma função que não retorna nada, ela fica representada assim em Assembly:

--------GCC/MSVC (x86):

f:
    ret 

(Apenas uma instrução (ret), que retorna a execução para quem chamou a função)

--------ARM:

f   PROC
    BX     lr
    ENDP

O endereço de retorno não é salvo no stack local ao usar o ARM, ele usa Link Register (LR), que seria um registrador que armazena o endereço no qual o computador tem que voltar depois que a função acabar. Ou seja, ao usar BX lr, ele termina o processo da função e retorna ao caller.

--------MIPS:

    j   $31
    nop

Tem dois métodos convencionais usados no MIPS quando são nomeados registradores: por número (de $0 a $31) ou por pseudo-nome ($V0, $A0, etc), o GCC (Compilador usado para transformar código C/C++ em MIPS Assembly e arquivos binários) utiliza de registradores com números, como o exemplo acima, e o IDA (Analiza binários de MIPS para gerar código de assembly legível, e gera código pseudo-C para reconstruir a lógica do programa) utiliza de registradores com pseudo-nomes, assim como no exemplo abaixo:

    j $ra
    nop

A primeira instrução é a de pular (J ou JR) que manda o flow da execução para o caller.

--------Outra função simples é a que retorna um valor constante:

int f(){

    return 123;

}

--------GCC/MSVC (x86):

f:
    mov     eax, 123
    ret

A primeira instrução move o valor 123 dentro do registrador EAX, que é geralmente um registrador utilizado para armazenar valor de retorno de uma função comum, e a segunda instrução retorna execução ao caller, e o caller, obviamente, pega o resultado da função pelo registrador EAX.

--------ARM:

f   PROC
    MOV     r0,#0x7b
    BX      lr
    ENDP

O ARM usa o registrador r0 para retornar os valores da função, assim como o EAX do GCC/MSVC, então 123 é armazenado no registrador r0. O mov, mesmo que dê a entender que seja mover, na verdade copia os dados dos quais a instrução for usada.

--------MIPS:

----GCC:

j   $31
li  $2,123

----IDA:

jr  $ra
li  $v0, 0x7B

O registrador para retorno de função do MIPS é o $2 ou $v0, e a instrução LI (Load Immediate) é equivalente ao MOV do ARM/x86. A instrução jr pula o fluxo para o caller, porém, diferentemente de outras arquiteturas, a linha após o jump sempre é executada no MIPS, por isso o número é retornado depois de jump, e isso se chama delay slot.

--------Agora, um código de C que apenas diz Hello World:

#include <stdio.h>

int main() {

    printf("Hello, world\n");
    return 0;

}

--------MSVC (x86):

CONST   SEGMENT ; começo de declaração de uma constante

$SG3830 DB      'hello, world', 0AH, 00H ; DB significa "define byte", ou seja, define o byte que vai representar a constante sendo declarada.

CONST   ENDS ; fim de uma declaração de uma constante

PUBLIC  _main ; deixa a função _main visível para o linker

EXTRN   _printf:PROC ; EXTRN diz para o assembler que a função existe, só que está em outro arquivo/biblioteca.

_TEXT   SEGMENT ; início do segmento de instruções

_main   PROC ; início da função _main

push    ebp ; salva o base pointer antigo na stack

mov     ebp, esp ; cria um novo stack frame

push    OFFSET $SG3830 ; empilha o endereço da string "hello, world" para o novo stack frame. OFFSET pega o endereço na memória, não o valor dela

call    _printf ; chama a função printf, usando o endereço da string de argumento

add     esp, 4 ; remove o argumento passado

xor     eax, eax ; eax = 0, que é o valor de retorno da função main. XOR (sim, o xor de exclusive or) é usado porque usa 2 bytes ao invés de 5 bytes que é usado na instrução MOV. Ou seja, o código visualmente parece complicado, mas no fim é só um uso de instruções diferentes pra buscar otimização

pop     ebp ; restaura o EBP antigo do stack frame inicial

ret     0 ; retorno da função, ou seja, volta para o caller

_main   ENDP ; fim da função _main

_TEXT   ENDS ; fim do segmento de instruções

O MSCVC usa a sintaxe Intel. Existem duas formas de escrever sintaxes no assembly x86, as sintaxes Intel e AT&T, onde o Intel é usado no MSVC (Microsoft Visual C++), e o AT&T é usado no Linux/Unix.

O compilador cria o arquivo l.obj, que está linkado ao l.exe. Nesse caso, o arquivo contém dois segmentos: CONST (para dados constantes) e _TEXT (para código). A string "hello, world" em C/C++ é de tipo const char[], porém esse tipo não tem um nome, então o compilador lida com o nome o chamando internamente de $SG3830.

O C/C++ sempre termina com um zero no final da string, que é representado no código Assembly acima, onde ao definir a constante, no final há o 00H, que representa o final da string por meio de um valor nulo (o byte 00000000). O \n é representado por 0AH nesse mesmo código, e vem antes do fim da string (00H), logicamente.

--------call = Instrução com intuito de executar uma função, pode receber argumentos ao serem escritos instruções de push anteriormente ao call (colocando argumentos no stack frame) tem um push invisível dando o endereço de caller, e depois um jump diretamente para a primeira linha da função que foi chamada pela própria instrução call. Ao dar push em mais de um argumento em uma função que necessita de mais argumentos, é necessário dar push em uma ordem invertida. Exemplo:

---Uma função chamada divideNums(dividendo, divisor) é criada, essa função deve ser chamada e executada com dois valores como argumento: o valor 10 que será o dividendo, e o valor 5 que será o divisor (ou seja, a ordem importa), logo, a ordem para dar push ao dar esse comando seria:

push 5
push 10
call divideNums

---Pois ao pensar em stack frame, é necessário pensar numa literal pilha de coisas, como pratos. Ao inserir um prato primeiro e depois outro, o outro prato será o primeiro prato a ser retirado, e então, o segundo prato a ser retirado será o primeiro que você colocou naquela pilha.

--------GCC (x86)

main    proc near

var_10  = dword ptr -10h

        push    ebp
        mov     ebp, esp
        and     esp, 0FFFFFFF0h
        sub     esp, 10h
        mov     eax, offset aHelloWorld ; "hello, world\n"
        mov     [esp+10h+var_10], eax
        call    _printf
        mov     eax, 0
        leave
        retn
main    endp

--------AND no GCC = É um "molde" (máscara de bits) para filtrar o endereço do dado que está no registrador que estiver sendo usado. Aonde estiver 1s, o dado passa, e aonde estiver 0s, o dado é convertido em zeros. Caso haja um endereço de 12 bits: 111000110111 no registrador ESP, ao moldar esse endereço para preservar os primeiros 6 bits e descartar o resto (transformar em 0), seria feito: "and esp, 0FC0h", logo o endereço ficaria: 111000000000. Essa instrução é feita a fim de facilitar ao processador de interpretar o endereço daquele dado no registrador, o tornando um múltiplo de 4, permitindo com que ele puxe o endereço do dado em 1 ciclo de viagem.

O resultado é quase o mesmo, o endereço da string é guardado no registrador EAX, e então salvo no stack frame. Além disso, o comando AND ESP, 0FFFFFFF0h alinha o endereço do valor no registrador ESP em um limite de 16 bytes, devido à eficiência do processador com alinhamento a múltiplos de 4.

O comando SUB ESP, 10h aloca 16 bytes no stack frame para manter o padrão do limite de 16 bytes, mesmo que apenas 4 bytes sejam necessários nessa situação. E depois a função printf() é chamada, assim que a string é alocada no ESP.

Na linha de MOV que vem logo após o SUB ESP, 10h, e a linha que coloca a string "hello world\n" no stack, vemos uma soma dentro de colchetes. O programa, nessa linha, basicamente pega o que estiver no EAX e joga para o esp +0, pois a var_10 é igual a -10h, e ao somar +10h com -10h, resulta em 0, e seria possível deixar mais direto ao ponto usando o 0, mas essa soma é feita para facilitar a vida de quem observa milhares de linhas de código Assembly (futuramente você, leitor, e eu também), utilizando de variáveis para demonstrar mais facilmente o que está sendo feito. O esp está em colchetes, pois o programa quer mostrar ao computador que este valor está na memória RAM, e não na CPU, e que esse valor não pode ser alterado.

Diferentemente do MSVC, o GCC usa MOV, ao invés de XOR que é usado no MSVC, quando ambos estão sem otimização ligada.

O comando leave faz automaticamente a função de retornar o ESP e o EBP para o início.
