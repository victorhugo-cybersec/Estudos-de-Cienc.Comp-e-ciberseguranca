--------------------Os programas do MySQL

---------------O que é o MySQL?

----------O MySQL é um pacote de ferramentas de linha de comando trabalhando juntas, essas ferramentas são divididas em categorias pela própria documentação do MySQL.

--------------------6.1 Visão Geral de Programas MySQL

-----"--help" = uma flag que pode ser utilizada para entender como uma ferramenta pode ser usada. Tem em maioria das ferramentas, como o comando "mysql".

-----mysqld = mysqld (MySQL Daemon, ou seja, o servidor MySQL) é essencial de estar rodando para utilizar de ferramentas cliente, pois os clientes ganham acesso aos bancos de dados conectando pelo servidor.

-----mysqld_safe = É um script gerenciador, ele é executado para iniciar o servidor MySQL desejado, e continua sendo executado de fundo, a fim de tentar rodar o servidor novamente se ele cair.

-----mysql.server = É considerado como um motor de partida para iniciar o mysqld (o servidor). Um script utilizado somente em SOs antigos que usam o sistema V (um sistema que determina o que será executado a cada estado do computador, seja desligando, ligado, reiniciando, etc), esse script rodaria no momento em que o computador ligasse, e ele puxa o mysqld_safe para ser executado após observar algumas configurações básicas e então se encerra. Resumidamente uma maneira de executar o servidor MySQL a partir do momento em que o computador for ligado nesses SOs mais antigos que não utilizam systemd (um sistema de inicialização para SOs que, no contexto do MySQL, já faz a função de ambos o mysql.server e o mysqld_safe).

-----mysqld_multi = Um script usado para situações específicas onde é desejado rodar mais de um servidor MySQL que estejam em portas TCP/IP e diferentes arquivos .sock Unix, e para cada um deles, é aberto o mysqld_safe, e cada servidor vai ter sua instância mysqld_safe aberta e rodando para previnir caídas do servidor.

----------Comandos (programas clientes) usados no MySQL:

-----mysql = Ferramenta de linha de comando utilizada pelo programa MySQL para receber comandos SQL ou executá-los de um arquivo no modo batch (de lote, ou seja, execução rápida, otimizada e automática, sem interação humana e com um output "feio", na intenção de ser eficiente).

-----mysqladmin = cliente que performa operações administrativas, como criar ou deletar bancos de dados, recarregar tabelas e mandar tabelas para o disco, reabrir arquivos log. Também pode ser usado para obter a versão, processo e informação de status do servidor.

-----mysqlcheck = um cliente que faz manutenção de tabelas, que observa, repara, analiza e otimiza tabelas.

-----mysqldump = cliente que transforma um banco de dados MySQL em um arquivo SQL, de texto, ou XML.

-----mysqlimport = cliente que faz, de maneira não literal, o inverso do cliente mysqldump, importando arquivos de texto em suas tabelas respectivas usando LOAD DATA.

-----mysqlsh = é um cliente avançado, um editor de código para o servidor MySQL, assim como o cliente mysql, ele provide a capacidade de criar scripts em javascript e python, e inclui APIs para trabalhar com MySQL.

-----mysqlshow = cliente que mostra informações sobre bancos de dados, tabelas, colunas e índices.

-----mysqlslap = cliente feito para emular carregamento cliente de um servidor MySQL, reportando o tempo de cada estágio. Funciona como se múltiplos clientes estivessem acessando o servidor.

-----Há vários outros comandos, mas nem todos são necessários de serem decorados ou anotados, quando for necessário alguma ferramenta que faça algo em específico, basta eu (você também) pesquisar.

--------------------Chapter 5 Tutorial

----------5.1 Conectando e desconectando do servidor.

-----Para conectar ao servidor, geralmente é necessário prover um nome de usuário MySQL ao usar "mysql", e muito provavelmente uma senha. Se o servidor roda numa máquina que não seja a que você está logando, é também necessário prover um nome de host. Ao saber tais parâmetros, é possível conectar assim: "mysql -h host -u user -p" (isso é uma template, não é necessário a parte do -h host caso o MySQL já estiver na máquina na qual você está usando. Ao usar a flag -p, não se coloca a senha depois, é necessário aparecer uma zona de input escrita "Enter password:" a fim de esconder a sua senha visualmente. Caso o MySQL na sua máquina permita isso, você pode entrar de maneira anônima no banco de dados, usando apenas "mysql").

-----Assim que o log-in dê certo, vai aparecer algo como mysql>, onde você poderá começar a criar e deletar tabelas e bancos de dados do jeito que quiser.

-----Para sair do log-in, basta digitar QUIT e dar enter, ou usar Ctrl + D caso o seu sistema seja Unix.

----------5.2 Colocando Queries

-----Ao entrar no servidor, você não estará em nenhum banco de dados, é como você estivesse no root do sistema e ainda precisasse realizar comandos para acessar um diretório, mas nesse caso, um banco de dados. No momento a documentação prefere não mostrar diretamente sobre a criação de banco de dados, a fim de dar um passo atrás do outro primeiramente.

-----É possível usar este query para verificar a versão do seu MySQL: "SELECT VERSION(), CURRENT_DATE;"

-----É visto que há a necessidade de colocar um ; no final das queries (Exceto em algumas, como o QUIT, por exemplo), e que, bom, quase tudo é em Caps Lock ativado, o que não é necessariamente algo para fazer o SQL funcionar, mas sim uma regra não escrita pelos usuarios de SQL, pois com letras maiúsculas para comandos e minúsculas para argumentos e outras coisas que serão estudadas futuramente, o código fica melhor de ser lido.

-----Assim como um shell comum, ao enviar as queries, o mysql executa a query e então mostra novamente "mysql>" pronto para receber novas queries. Seus outputs quase sempre serão em formato de tabela (colunas e linhas) e a primeira linha sempre será o nome das colunas que você coleta de tabelas de bancos de dados. Se você resgatar o valor de uma expressão ao invés de uma coluna de uma tabela (como no exemplo visto), o mysql nomeia a coluna usando a expressão em si.

-----Como um dos focos do MySQL é ser otimizado, ele mostra quantas linhas foram retornadas e quão longo demorou para a query ser executada, o que dá uma noção básica da performance do servidor. Os valores são imprecisos por representarem o tempo de um relógio literal, e não da CPU ou da máquina, além de serem afetados por carregamento de server e a latência da rede.

-----O MySQL pode ser utilizado como uma calculadora também, ao ser usado esse query, por exemplo: "SELECT (20+4)/32;".

-----É possível fazer mais de uma linha de comando, apenas se oportunando do ; que representa o fim de uma linha de query (Exemplo: "SELECT VERSION(); SELECT NOW();").

-----As queries não precisam ser feitas em 1 única linha. É possível estender a query para várias linhas apenas dando enter antes de terminar o comando com ponto e vírgula.

Exemplo:
SELECT
VERSION()
,
NOW()
;

Ao fazer isso, o mysql> se torna -> a cada linha nova, a fim de mostrar ao usuário que a query não foi fechada. Na verdade, há muitas maneiras que o MySQL usa para se comunicar com o usuário do programa através do prompt:

mysql> = Pronto para uma nova query.
-> = Esperando a próxima linha da query de múltiplas linhas.

'> = Esperando pela próxima linha, esperando o usuário fechar a string que começou com uma única aspa '. Nesse caso, basta utilizar '\c, o que é interpretado pelo SQL como um caractere, fechando a string.

"> = Quase a mesma coisa de '>. Está esperando pela próxima linha, mas esperando também o usuário fechar a string que começou com aspas duplas ". Nesse caso, basta utilizar "\c, o que é interpretado pelo SQL como um caractere, fechando a string.

`> = Esperando pela próxima linha, esperando o usuário fechar um identificador que começou com um acento grave `. Nesse caso, basta utilizar `\c, o que é interpretado pelo SQL como um caractere, fechando a string.

/*> = Esperando pela próxima linha, esperando o usuário fechar o comentário que começou com /*.

----------5.3 Criando e usando um banco de dados.

-----SHOW DATABASES; =

Para exibir bancos de dados, é possível utilizar a query "SHOW DATABASES;". É importante falar que essa query pode mostrar bancos de dados variados dependendo das suas permissões. Caso você não tenha a permissão de "SHOW DATABASES", você não irá ter acesso a poder ver alguns bancos de dados.

-----USE *banco de dados* = 

A instrução "USE" (não é uma query por não consultar ou alterar dados diretamente) deve ser usado em uma linha e não necessita de ponto e vírgula no final. Ele é usado para entrar em um banco de dados, então, caso haja por exemplo, um banco de dados chamado teste, é possível entrar nele usando a query: "USE TEST".

-----CREATE DATABASE *nome do banco*; = 

Para criar um banco de dados, você pode usar o comando "CREATE DATABASE" e depois o nome do banco de dados que vai ser criado. É importante saber que nomes de bancos de dados são case-sensitive em sistemas Unix (Letras maiúsculas e minúsculas tem diferença). Para o Windows, ainda assim é recomendada a prática de usar letras maiúsculas e minúsculas de acordo com o nome do banco de dados qual referenciado.

Ao criar um banco de dados, você não entra automaticamente nele. Logo, é bom usar o comando "USE", assim como todas as vezes que você começar uma sessão mysql, é necessário entrar no banco novamente.

-----SHOW TABLES; =

Com esse comando, é possível ver as tabelas existentes no banco de dados em que você estiver.

-----Na documentação, é explicado que a parte mais difícil de criar uma tabela é a parte de definir as colunas e linhas, tendo em mente que é necessário dar dados que não precisam ser manualmente alterados, e que sejam relevantes.

-----É dado um exemplo de um banco de dados que armazena informações de pets, e em uma das colunas, é armazenado um dado de dia de nascença, o MySQL provide cálculos com dias, então com o dia de nascimento do pet, é possível calcular automaticamente a idade do pet e não haverá a necessidade de colocar uma coluna de idade onde é necessária ser alterada a cada ano que se passa. Ao criar a tabela de pets, foi usada essa query:

CREATE TABLE pet (name VARCHAR(20), owner VARCHAR(20),
        species VARCHAR(20), sex CHAR(1), birth DATE, death DATE);

-----CREATE TABLE *dados*; = Um comando usado para criar tabelas dentro de um banco de dados.

No exemplo acima, vemos o nome da tabela que está sendo criada, e logo após, um parênteses é aberto, nele são dadas informações sobre o nome das colunas e o conteúdo que pode ser incluido nelas.
O "name VARCHAR(20)" cria uma coluna de nome "name" que pode receber um texto de no máximo 20 caracteres.
"owner VARCHAR(20)" é o mesmo conceito da coluna "name", mas com nome diferente.
"species VARCHAR(20)" a mesma coisa das colunas "owner" e "name", mas com nome diferente.
"sex CHAR(1)" cria uma coluna com nome "sex" e que só aceita um caractere de texto, que nesse contexto deveria receber ou a letra F ou a letra M.
"birth DATE" cria uma coluna chamada birth que pode receber um dia como dado, é aonde o dia de aniversário é inserido, e cálculos podem ser feitos de acordo com o dia atual. É importante dizer que o MySQL lê datas com essa template: YYYY-MM-DD, ou seja, ano, mês e dia, respectivamente.
"death DATE" é a mesma coisa da coluna "birth", mas pode receber valor nulo caso o pet esteja vivo, logicamente.

-----DESCRIBE *tabela*; = Exibe a estrutura da tabela que for citada.
O resultado do comando "DESCRIBE" com a tabela do exemplo lá de cima dá nisso:

+---------+-------------+------+-----+---------+-------+
| Field   | Type        | Null | Key | Default | Extra |
+---------+-------------+------+-----+---------+-------+
| name    | varchar(20) | YES  |     | NULL    |       |
| owner   | varchar(20) | YES  |     | NULL    |       |
| species | varchar(20) | YES  |     | NULL    |       |
| sex     | char(1)     | YES  |     | NULL    |       |
| birth   | date        | YES  |     | NULL    |       |
| death   | date        | YES  |     | NULL    |       |
+---------+-------------+------+-----+---------+-------+

É possível ver que há colunas com valores e nomes diferentes, isso vai ser aprofundado futuramente.

----------5.3.3 Carregando dados numa tabela.

-----Utilizando ainda o exemplo da tabela de pets citada anteriormente, para "popular" uma tabela vazia, geralmente é usado um arquivo .txt com todas as linhas de comando usadas para criar várias instâncias na tabela. Algo interessante é que, com o comando que já será explicado, "LOAD", é possível fazer um arquivo .txt apenas com os dados nos lugares respectivos de cada coluna e um tab separando cada dado, e na hora de deixar um valor nulo, é usado o \N, o que resulta em algo mais ou menos assim dentro do arquivo:

Whistler        Gwen    bird    \N      1997-12-09      \N
(name)          (owner) (species,sex)   (birth)         (death)
